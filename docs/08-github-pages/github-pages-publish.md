
# GitHub Pages: публикация сайта и обновление документации

## Назначение

GitHub Pages используется для публикации сайта документации, созданного с помощью MkDocs Material.  
Сайт автоматически обновляется при выгрузке изменений в репозиторий GitHub и доступен по публичной ссылке.

Использование GitHub Pages позволяет:
- продемонстрировать результат проектной работы;
- предоставить доступ преподавателю и участникам команды;
- хранить документацию вместе с исходным кодом проекта.

---

## Подготовка проекта к публикации

Для публикации сайта в репозитории должны присутствовать:

- папка `docs/` с файлами `.md`;
- файл `mkdocs.yml` в корне проекта;
- файл `README.md` с описанием проекта.

Пример структуры проекта:

```text
project-root/
├─ docs/
│  ├─ assets/
│  ├─ index.md
│  └─ ...
├─ mkdocs.yml
└─ README.md
```

---

## Выгрузка обновлений проекта на GitHub

Перед публикацией сайта необходимо зафиксировать и отправить изменения в репозиторий.

В терминале, находясь в корне проекта, выполните:

```bash
git status
git add .
git commit -m "Update documentation"
git push -u origin main
```

После выполнения этих команд изменения будут загружены в удалённый репозиторий GitHub.
