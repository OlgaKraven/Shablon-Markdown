    Создание структуры проекта MkDocs

    Назначение

    Перед началом работы с документацией необходимо создать корректную структуру проекта, соответствующую требованиям MkDocs и теме Material for MkDocs.
    Правильно организованная структура: - упрощает навигацию; - обеспечивает корректное подключение ресурсов (стилей, скриптов, изображений); - соответствует требованиям учебной и проектной документации.

    Базовая структура проекта

    В корне проекта должны находиться следующие элементы:

    project-root/
├─ docs/
│  ├─ assets/
│  ├─ index.md
│  └─ ...
├─ mkdocs.yml
└─ README.md

    Назначение элементов

    • docs/ — основная папка с Markdown-документацией;
    • mkdocs.yml — конфигурационный файл MkDocs;
    • README.md — описание проекта для GitHub и участников команды.

    Структура папки docs/assets

    Внутри папки docs создаётся каталог assets, предназначенный для хранения вспомогательных файлов. Рекомендуемая структура:

    docs/
└─ assets/
   ├─ files/
   ├─ icons/
   ├─ images/
   ├─ js/
   └─ stylesheets/

    Назначение каталогов

    • assets/files/ Файлы для скачивания:

      • отчёты (.pdf, .docx);
      • архивы проектов (.zip);
      • методические материалы.

    • assets/icons/ Иконки проекта:

      • логотип сайта;
      • favicon;
      • иконки навигации.

    • assets/images/ Изображения для документации:

      • схемы;
      • диаграммы;
      • скриншоты VS Code и терминала.

    • assets/js/ Пользовательские JavaScript-файлы:

      • инициализация Mermaid (mermaid-init.js);
      • дополнительные скрипты при необходимости.

    • assets/stylesheets/ Пользовательские CSS-файлы:

      • extra.css — переопределение стилей темы;
      • оформление таблиц, диаграмм, цитат.

    Создание структуры в VS Code

    1. Откройте папку проекта в VS Code
    2. В панели Explorer создайте папку docs
    3. Внутри docs создайте папку assets

    4. В папке assets создайте каталоги:

      • files
      • icons
      • images
      • js
      • stylesheets

    5. В корне проекта создайте файлы:

      • mkdocs.yml
      • README.md

    Все действия рекомендуется выполнять через VS Code, чтобы структура проекта сразу отображалась корректно.

    Пример содержимого README.md

    # Учебная практика — документация MkDocs

Проект содержит учебную и проектную документацию, оформленную с использованием MkDocs Material.

## Запуск проекта

1. Установить зависимости
2. Выполнить команду:
   ```bash
      mkdocs serve
    ```

3. Открыть в браузере: [http://127.0.0.1:8000](http://127.0.0.1:8000)
